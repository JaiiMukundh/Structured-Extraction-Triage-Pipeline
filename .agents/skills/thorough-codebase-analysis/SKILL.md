# Thorough Codebase Analysis

**CRITICAL RULE:** Do not be a mere documentation or comment reader. You must look at and analyze the actual codebase and read files thoroughly. 
When instructed to read or process a file, process the *entire* file. Do not worry about token efficiency. 
Understand the project at a deep level and derive all findings empirically from the code and full text, rather than relying on summaries or assumptions.
